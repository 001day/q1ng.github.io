0x02 利用 SSH 公钥免密登录

如果目标服务器是root权限，那么就可以通过未授权访问，可以将自己的ssh公钥写入 /root/.ssh/authorized_keys 。如果对方redis服务器有密码，我们直接可以 ssh root@目标IP 登录，无需密码 （条件在于对方开了ssh服务）

原理在于

Linux 服务器的 SSH 服务（sshd）通常有两种验证你身份的方式：

  • 方式一：密码认证（Password）
    • 服务器问：“口令是什么？”
    • 你回答：“123456”。
    • 服务器比对正确，开门。
  • 方式二：公钥认证（Public Key）—— 也就是“免密登录”
    • 服务器不问密码，而是去检查一个特定的文件：~/.ssh/authorized_keys。
    • 这个文件就像一份 “白名单”，里面记录了所有允许免密进入的公钥。
    • 逻辑是： “谁拥有这把公钥对应的私钥，谁就可以直接进来，不用密码。”

总结

对于redis服务器处于内网，我们就可以利用ssrf实现未授权访问（暴力点就是直接内网渗透了）；如果在公网上存在一个服务器监听6379端口的redis服务器，那么就是送福利了哈哈
